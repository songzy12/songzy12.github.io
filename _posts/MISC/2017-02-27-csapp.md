## bit, byte, word

A bit is a BInary digiT. So a bit is a zero or a one.

A byte is a sequence of bits. Since the mid 1960's a byte has been 8 bits in length. Since the number of things that you can enter on a computer keyboard is smaller than 256, a code for a keystoke is represented with a code within a byte.

A word is the number of bits that are manipulated as a unit by the particular CPU of the computer. Data is fetched from memory to the processor in word size chunks and manipulated by the ALU in word size chunks.

[Reference](http://www.cs.scranton.edu/~cil102/data_bits.html).

## 32 bit, 64 bit

The terms 32-bit and 64-bit refer to the way a computer's processor (also called a CPU), handles information. 

Size of a pointer should be 8 byte on any 64-bit C/C++ compiler, but not necessarily size of int.

In many programming environments for C and C-derived languages on 64-bit machines, int variables are still 32 bits wide, but long integers and pointers are 64 bits wide. These are described as having an LP64 data model. Another alternative is the ILP64 data model in which all three data types are 64 bits wide, and even SILP64 where short integers are also 64 bits wide.

[Reference](https://en.wikipedia.org/wiki/64-bit_computing)
